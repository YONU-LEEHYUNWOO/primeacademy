# 📘 PRD (제품 요구사항 문서): 수학학원 랜딩페이지

## 📌 1. 제품 개요 (Product Overview)

**제품명:** 프라임 수학학원 랜딩페이지
**타깃 사용자:** 학부모 (중·고등학생 자녀를 둔 40~50대 중심)
**플랫폼:** Web (반응형)
**저장 방식:** Local (DB 없음, 단순 Form Submit 또는 이메일 전송 기반)

### 🎯 목적

학부모들이 학원의 강점, 교육 철학, 성과를 신뢰감 있게 확인하고
**온라인 방문 상담 예약으로 전환**하도록 유도하는 목적의 랜딩페이지

## 💻 7. 기술 스택 제안 (Technical Stack Recommendations)

| 항목                      | 기술                                     | 설명                | 선택 이유             |
| ----------------------- | -------------------------------------- | ----------------- | ----------------- |
| **Frontend Framework**  | **Next.js (Static Export)**            | 정적 사이트 생성 (SSG)   | 빠른 로드 속도, SEO 최적화 |
| **UI Framework**        | **TailwindCSS**                        | Utility-first CSS | 일관된 반응형 디자인       |
| **Form Handling**       | **EmailJS / Google Form**              | 이메일 기반 폼 제출       | DB 불필요, 간단한 관리    |
| **Chart Visualization** | **Chart.js / Recharts**                | 통계 시각화            | 학원 성과 데이터 시각화     |
| **Map Integration**     | **Google Maps Embed API**              | 위치 정보 표시          | 직관적 위치 확인         |
| **Hosting**             | **Vercel / Netlify**                   | 정적 호스팅            | 빠른 배포, 유지보수 간소화   |
| **Analytics**           | **Google Analytics / Naver Analytics** | 유입 및 클릭 분석        | 마케팅 효율 측정         |

## ✅ 결론

본 수학학원 랜딩페이지는 **“학부모 신뢰 중심의 데이터형 랜딩페이지”**로,
정보 전달보다 **신뢰·성과·전환(예약)** 중심의 UX를 구축합니다.

Next.js 기반의 정적 페이지 구조로 **빠른 로딩과 SEO 최적화**,
TailwindCSS를 통한 **반응형 심플 디자인**,
그리고 **DB 없이도 운영 가능한 예약 폼**을 통해
**운영 효율성과 실용성을 극대화한 홍보용 웹 페이지**로 설계됩니다.
