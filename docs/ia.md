# 🗂️ 정보구조도 (IA: Information Architecture)

**프로젝트명:** 프라임 수학학원 랜딩페이지
**플랫폼:** Web (반응형)
**내비게이션 형태:** Topbar
**인증 여부:** 없음 (공개형 랜딩페이지)

---

## 📘 1. 사이트 맵 (Site Map)

```
홈 (/)
├── 학원 소개 (/about)
├── 성과 및 통계 (/results)
├── 강사진 소개 (/teachers)
├── 차별점 소개 (/difference)
├── 오시는 길 (/location)
└── 상담 예약 (/reservation)
```

🔹 **설명:**

* 모든 페이지는 **Topbar** 내 링크를 통해 접근 가능
* 인증, 로그인 기능 없음 (단일 사용자 경험 중심)
* 상담 예약 섹션은 별도 페이지 및 메인 CTA로 중복 배치

---

## 🧭 2. 사용자 흐름 (User Flow)

### 🎯 주요 사용자 시나리오: “학부모가 학원 정보를 확인하고 상담을 예약하는 과정”

1️⃣ **홈 진입**
→ 브랜드 신뢰 문구와 대표 이미지 확인
→ CTA “상담 예약하기” 버튼 노출

2️⃣ **스크롤 탐색**
→ 학원 소개 → 통계 → 차별점 → 강사진 소개 → 위치 정보
→ 각 섹션에서 “예약하기” 버튼 반복 배치

3️⃣ **상담 예약 클릭**
→ 예약 폼 페이지로 이동 (`/reservation`)
→ 이름, 연락처, 희망 날짜 입력

4️⃣ **폼 제출 후 확인 메시지 표시**
→ “상담 신청이 완료되었습니다. 담당자가 연락드리겠습니다.”

5️⃣ **페이지 종료 or 재방문 유도**

## 🧱 3. 내비게이션 구조 (Navigation Structure)

| 위치              | 구성 요소                            | 설명        | 동작 방식               |
| --------------- | -------------------------------- | --------- | ------------------- |
| **Topbar**      | 로고 / 학원 소개 / 성과 / 강사진 / 차별점 / 예약 | 주요 페이지 접근 | 모든 페이지에서 고정 표시      |
| **Hero CTA 버튼** | “상담 예약하기”                        | 주요 전환 버튼  | `/reservation`으로 이동 |
| **Footer**      | 학원명, 주소, 연락처, 이용약관               | 정보 제공 영역  | 모든 페이지 공통 표시        |

🔹 **모바일 대응:**
Topbar는 햄버거 메뉴(`☰`)로 전환되며, 클릭 시 슬라이드 메뉴로 모든 항목 표시

## 📂 5. 콘텐츠 구성 (Content Organization)

| 페이지         | 주요 콘텐츠 블록                            | 설명                   |
| ----------- | ------------------------------------ | -------------------- |
| **홈**       | Hero 배너 / CTA / 학원 핵심 메시지 / 하단 예약 버튼 | 브랜드 신뢰 확보 및 즉시 전환 유도 |
| **학원 소개**   | 교육 철학 / 커리큘럼 / 관리 시스템 이미지            | 학원 철학 및 학습 접근 방식 소개  |
| **성과 및 통계** | 합격률 / 향상률 그래프 / 후기                   | 객관적 수치와 신뢰 확보        |
| **차별점 소개**  | 커리큘럼 특징 / 학습 관리 프로세스 / 타 학원 비교       | 차별성 강조               |
| **강사진 소개**  | 프로필 카드 / 학력 / 경력 / 수업 철학             | 신뢰도 및 전문성 강화         |
| **오시는 길**   | Google Maps / 주소 / 연락처 / 운영시간        | 물리적 위치 안내            |
| **상담 예약**   | 입력 폼(이름, 연락처, 일정, 문의내용) / 제출 확인 메시지  | 전환의 최종 단계            |

## 💬 6. 인터랙션 패턴 (Interaction Patterns)

| 사용자 행동           | 시스템 반응               | 피드백 방식                 |
| ---------------- | -------------------- | ---------------------- |
| **Topbar 메뉴 클릭** | 해당 페이지로 스크롤 이동 또는 이동 | Smooth Scroll / 페이지 전환 |
| **CTA 버튼 클릭**    | 예약 폼 페이지 이동          | 페이지 전환 애니메이션           |
| **폼 입력 후 제출**    | 확인 메시지 표시            | 모달 팝업 or Toast 메시지     |
| **이미지 hover**    | 확대 및 설명 표시           | Transition 효과          |
| **모바일 메뉴 클릭**    | 슬라이드 메뉴 오픈           | Overlay 어둡게 처리         |

🔹 **접근성 고려 (A11y)**

* 모든 이미지에 대체 텍스트(`alt`) 추가
* 폼 입력 항목에 라벨(`label for=""`) 지정
* 색상 대비 WCAG 2.1 AA 준수

## 🌐 7. URL 구조 (URL Structure)

| 페이지         | URL 예시         | SEO 최적화 요소                                                                  |
| ----------- | -------------- | --------------------------------------------------------------------------- |
| **홈**       | `/`            | `<title>프라임 수학학원 - 성적 향상의 중심</title>`                                       |
| **학원 소개**   | `/about`       | `<meta name="description" content="프라임 수학학원의 교육 철학과 관리 시스템을 소개합니다.">`       |
| **성과 및 통계** | `/results`     | `<meta name="description" content="합격률, 성적 향상률 등 프라임 수학학원의 통계 정보를 확인하세요.">` |
| **차별점 소개**  | `/difference`  | `<meta name="description" content="다른 학원과 차별화된 프라임만의 커리큘럼을 만나보세요.">`        |
| **강사진 소개**  | `/teachers`    | `<meta name="description" content="프라임 수학학원의 전문 강사진을 소개합니다.">`              |
| **오시는 길**   | `/location`    | `<meta name="description" content="학원 위치, 연락처, 운영시간 안내입니다.">`               |
| **상담 예약**   | `/reservation` | `<meta name="description" content="상담 예약을 통해 자녀에게 맞는 학습 상담을 받아보세요.">`       |

🔹 **SEO 전략:**

* 제목(`H1`)은 각 페이지 고유 키워드 기반
* 이미지 파일명은 한글 대신 영문(`teacher-profile-01.jpg`) 사용
* Schema.org의 `LocalBusiness` 구조화 데이터 적용

## 🧩 8. 컴포넌트 계층 구조 (Component Hierarchy)

```
<AppLayout>
 ├── <Header>
 │    ├── <Logo />
 │    ├── <NavMenu />
 │    └── <CTAButton />
 ├── <Main>
 │    ├── <HeroSection />
 │    ├── <AboutSection />
 │    ├── <StatsSection />
 │    ├── <DifferentiationSection />
 │    ├── <TeachersSection />
 │    ├── <LocationSection />
 │    └── <ReservationForm />
 └── <Footer>
      ├── <ContactInfo />
      ├── <PolicyLinks />
```

| 컴포넌트                | 역할               | 반응형 동작                   |
| ------------------- | ---------------- | ------------------------ |
| **Header**          | 내비게이션 고정, CTA 포함 | 모바일 시 햄버거 메뉴 전환          |
| **HeroSection**     | 브랜드 첫인상 및 예약 CTA | 배경 이미지 높이 80vh, 모바일 시 축소 |
| **StatsSection**    | Chart.js 그래프 표시  | 그래프 세로 방향 재배치            |
| **TeachersSection** | 강사 카드 리스트        | 모바일에서는 1열, 데스크톱에서는 3열    |
| **ReservationForm** | 상담 예약 폼          | 입력 필드 세로 정렬, 모바일 키보드 대응  |
| **Footer**          | 주소/연락처 제공        | 2열 → 1열 전환 (mobile)      |

## 📱 9. 반응형 디자인 고려사항 (Responsive Design Considerations)

| 디바이스                    | 해상도              | 레이아웃 변화      |
| ----------------------- | ---------------- | ------------ |
| **Mobile (≤768px)**     | 단일 컬럼, Topbar 축소 | CTA 버튼 하단 고정 |
| **Tablet (768~1024px)** | 2컬럼 구성, 텍스트 확대   | 이미지 비율 유지    |
| **Desktop (≥1024px)**   | 3컬럼, 넓은 여백 구성    | Hero 텍스트 좌정렬 |
| **Wide (≥1440px)**      | 최대폭 1440px 고정    | 콘텐츠 중앙 정렬    |

## ✅ 결론

본 IA 문서는 **학부모 대상의 전환 중심 랜딩페이지**를 위한 정보 구조로,
사용자가 **신뢰 기반의 정보를 빠르게 파악하고 상담 예약으로 이동하도록 최적화**되었습니다.

* **Topbar 기반 단순 내비게이션**으로 탐색 피로도 최소화
* **섹션형 구조 + 반복 CTA**로 전환율 극대화
* **반응형 및 접근성 표준 준수**로 모바일 친화적 UX 확보

👉 이 구조는 **Next.js + TailwindCSS 기반 정적 페이지** 구현 시 가장 효율적인 사용자 여정을 제공합니다.
